## HTTP-статусы

| Код | Статус | Когда возникает |
| --- | ------ | --------------- |
| 200 | OK | Успешное получение или обновление ресурса |
| 201 | Created | Успешное создание ресурса |
| 400 | Bad Request | Ошибка валидации: отсутствуют обязательные поля или неверный формат |
| 404 | Not Found | Ресурс с указанным `id` не найден |
| 409 | Conflict | Конфликт состояния — см. коды ошибок ниже |
| 500 | Internal Server Error | Внутренняя ошибка сервера |

## Коды ошибок в теле ответа

При ошибке API возвращает JSON с полями `code` и `message`:

```json title="Пример ответа 409"
{
  "code": "HERO_NOT_AVAILABLE",
  "message": "Герой недоступен: статус on_mission"
}
```

| Код ошибки | HTTP | Описание |
| ---------- | ---- | -------- |
| `VALIDATION_ERROR` | 400 | Не переданы обязательные поля или нарушены ограничения |
| `HERO_NOT_FOUND` | 404 | Герой с указанным `hero_id` не найден |
| `INCIDENT_NOT_FOUND` | 404 | Инцидент с указанным `incident_id` не найден |
| `TASK_NOT_FOUND` | 404 | Задача с указанным `task_id` не найдена |
| `HERO_NOT_AVAILABLE` | 409 | Герой не может быть назначен: статус `on_mission` или `resting` |
| `INCIDENT_NOT_OPEN` | 409 | Инцидент уже имеет назначенного героя или закрыт |
| `DUPLICATE_HERO` | 409 | Герой с таким именем уже зарегистрирован |
