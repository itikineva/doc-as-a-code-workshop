## Front matter

Каждый файл начинается с блока front matter в формате YAML, ограниченного `---`.

```yaml title="Пример front matter"
---
title: Название страницы
sidebar_position: 1
description: Краткое описание для SEO и превью в поиске
---
```

| Поле | Обязательно | Описание |
| ---- | ----------- | -------- |
| `title` | Да | Заголовок страницы — отображается в sidebar и вкладке браузера |
| `sidebar_position` | Да | Порядок страницы внутри раздела (число) |
| `description` | Рекомендуется | Краткое описание для поисковых систем и hover-подсказок |

:::warning
Поле `title` в front matter заменяет заголовок H1. Не нужно дублировать его на странице через `# Название`.
:::

## Заголовки

Заголовки формируют структуру страницы и автоматически попадают в правую навигацию (Table of Contents).

| Уровень | Markdown | Назначение |
| ------- | -------- | ---------- |
| H1 | `# Заголовок` | Не использовать — заменяется полем `title` в front matter |
| H2 | `## Раздел` | Основные разделы страницы |
| H3 | `### Подраздел` | Подразделы внутри H2 |
| H4 | `#### Пункт` | Использовать редко, только если структура действительно требует 4 уровня |

**Правила:**

- Не пропускать уровни: после `##` должен идти `###`, а не `####`
- Каждый заголовок должен быть окружён пустыми строками
- Заголовки не должны заканчиваться точкой

## Имена файлов

- Формат: `kebab-case` (строчные буквы, слова через дефис)
- Символы: только латиница, цифры и дефис
- Расширение: `.md`

```
✓  scenario1.md
✓  tech-stack.md
✓  error-codes.md

✗  Scenario1.md      (заглавная буква)
✗  техСтек.md        (кириллица)
✗  error_codes.md    (подчёркивание)
```

## Имена папок

Те же правила, что для файлов. Имя папки влияет на URL, но не отображается в sidebar напрямую — отображаемое название задаётся в `_category_.yml`.

## Файл _category_.yml

Файл `_category_.yml` управляет отображением раздела в sidebar. Его нужно создать в папке, если нужно задать заголовок, порядок или описание категории.

```yaml title="docs/scenario/_category_.yml"
label: Сценарии использования
position: 4
collapsible: true
collapsed: false
```

| Поле | Обязательно | Описание |
| ---- | ----------- | -------- |
| `label` | Да | Название категории в sidebar |
| `position` | Рекомендуется | Порядок категории среди других разделов (число) |
| `collapsible` | Нет | Можно ли сворачивать раздел (по умолчанию `true`) |
| `collapsed` | Нет | Свёрнут ли раздел при открытии страницы (по умолчанию `true`) |
| `link.type` | Нет | Тип ссылки заголовка категории: `generated-index` (авто-индекс) или `doc` (конкретная страница) |

### Автоматический индекс категории

Если задать `link.type: generated-index`, Docusaurus сам создаёт страницу-индекс со списком всех страниц раздела:

```yaml title="docs/arch/_category_.yml"
label: Архитектура
position: 2
link:
  type: generated-index
  description: Контекстная и контейнерная диаграммы системы
```

### Ссылка на конкретную страницу

Если раздел открывается с определённой страницы — используй `link.type: doc`:

```yaml title="docs/api/_category_.yml"
label: API Reference
position: 3
link:
  type: doc
  id: api/herotask
```

:::note
Значение `id` — это путь к `.md`-файлу относительно папки `docs/`, без расширения. Например, файл `docs/api/herotask.md` имеет id `api/herotask`.
:::

:::tip
Без `_category_.yml` Docusaurus отображает папку в sidebar, используя её имя как метку и не показывает страницу при клике на заголовок категории.
:::
